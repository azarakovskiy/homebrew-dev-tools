## Working agreements
- For ambiguous or risky tasks, use the planner first.
- When the same mistake happens twice, propose a skill or AGENTS.md improvement.

## Always
- Keep work minimal and scoped to the request.
- Iterate in explicit, small, reviewable steps.
- Prefer simple maintainable solutions over speculative abstraction.

## Never
- use dev/local paths (e.g. `localhost` or local file path) in production code or text

## Conflict resolution
- Explicit user request and project-local instructions override global baseline. Flag it when this happens.
- Prefer simpler and more readable solution when two options are valid.
- Keep changes minimal, reversible, and easy to review.
